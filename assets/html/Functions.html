function modifier looks just like a function, but uses the keyword modifier instead of the keyword function. And it can't be called directly like a function can â€” instead we can attach the modifier's name at the end of a function definition to change that function's behavior.

modifiers are added as an extension before the function body, and they themselves can have arguments passed in. For the common use case as a requirements check on function access, a modifier could be added with the user's age passed in, to confirm age and access to the function.

 - one of the most common use-cases is to add a quick require check before a function executes.

Syntax is as below:

modifier <modifierName>(<input parameters>) {
  _;
}

OnlyOwner

modifier onlyOwner {
  require(msg.sender == owner);
  _;
}

In this example, the onlyOwner modifier can be applied to a function to ensure that only the contract owner is able to execute the function. The require statement checks that the msg.sender (the address of the message caller) is equal to the contract's owner. If the condition is not met, the function execution will be stopped.

The onlyOwner keyword is added after the function type definition and before function body.

mapping (uint => uint) public age;
// Modifier that requires this user to be older than a certain age:
modifier olderThan(uint _age, uint _userId) { require(age[_userId] >= _age); _; }
// Must be older than 16 to drive a car (in the US, at least).
// We can call the `olderThan` modifier with arguments like so:
function driveCar(uint _userId) public olderThan(16, _userId) { // Some function logic }
