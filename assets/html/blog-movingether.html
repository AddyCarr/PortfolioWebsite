  • Being able to inherintly move money inside the envelope of functions is a major point of advantage for solidity.
  • msg.value - amount of eth sent to the contract
  • address.transfer(value) - transfers ether to address

- address(this).balance - returns balance stored on contract

Payable example

contract OnlineStore { function buySomething() external payable {
// Check to make sure 0.001 ether was sent to the function call:
require(msg.value == 0.001 ether);
// If so, some logic to transfer the digital item to the caller of the function:
transferThing(msg.sender); } }

Withdawal example

contract GetPaid is Ownable { function withdraw() external onlyOwner { address payable _owner = address(uint160(owner())); _owner.transfer(address(this).balance); } }
  • Note that withdrawls often use Ownable, and hence this must be imported beforehand.

Variables can be marked payable aswell. if you want to transfer out of a variable, it can be marked payable within a function like the below:

payable(msg.sender).transfer()
